## Overview

**first-native** is a React Native productivity app built with Expo (SDK 54). It provides three core modules:

1. **Home** — Calendar-centric task view with daily tasks and upcoming items
2. **Notes** — Hierarchical folder/note system with rich text editing (TipTap)
3. **Tasks** — Task management with deadlines and note linking

The app uses a client-side architecture with Zustand for state management and no backend integration (currently using dummy data).

## Components

### TipTap Editor (`components/tiptap-editor/`)
- **Purpose:** Rich text note editing with WebView-based TipTap integration
- **Features:** Headings, bold/italic, lists, blockquotes, links, YouTube embeds, LaTeX math
- **Pattern:** WebView with injected HTML/JS, communicates via `postMessage`
- **Dependencies:** `react-native-webview`, custom HTML bundle

### Calendar (`components/calendar.tsx`)
- **Purpose:** Month-view calendar for task date selection
- **Features:** Date selection, task indicators (dots), month navigation
- **Used by:** Home screen

### Notes Store (`stores/notes-store.ts`)
- **Purpose:** Centralized notes & folders state management
- **Features:** CRUD for notes/folders, soft delete, trash management, sorting
- **Pattern:** Zustand with typed interface
- **Data model:** Notes with `folderId`, Folders with `parentId` (nested hierarchy)

### Tasks Store (`stores/tasks-store.ts`)
- **Purpose:** Centralized tasks state management
- **Features:** CRUD, toggle complete, date filtering, sorting
- **Pattern:** Zustand with typed interface

## Data Flow

1. **Note Creation:**
   - User taps FAB → `createNote(folderId)` in store → Navigate to `/Notes/[id]`
   - Editor loads with blank content → User edits → Auto-save via `updateNote()`

2. **Task Viewing (Home):**
   - Home screen mounts → Subscribe to `useTasksStore.tasks`
   - Calendar date selection → Filter tasks by `deadline` date
   - Task tap → Navigate to `/Tasks/[id]` modal

3. **Folder Navigation (Notes):**
   - Notes list mounts → Gets folders/notes via `getNotesInFolder(currentFolderId)`
   - Folder tap → Set `currentFolderId` → Re-render list
   - Breadcrumb navigation for nested folders

## Conventions

**Naming:**
- Components: `kebab-case.tsx` files, `PascalCase` exports
- Hooks: `use-*.ts` prefix
- Stores: `*-store.ts` suffix

**Structure:**
- Screens in `app/` with Expo Router file-based routing
- Reusable components in `components/`
- State in `stores/` with Zustand

**Styling:**
- `StyleSheet.create()` for styles
- Theme colors from `@react-navigation/native`
- Dark/light mode via `useColorScheme`

**State:**
- Zustand for global state
- Local state with `useState` for component-specific UI
- No Redux, no Context API for data
